# Security Policy

## Supported Versions

This project is a learning resource and is continuously updated. We recommend using the latest version.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

We take the security of this project seriously. If you discover a security vulnerability, please follow these steps:

### How to Report

1. **Do NOT** open a public issue
2. **Do NOT** disclose the vulnerability publicly until it has been addressed
3. Email the details to the repository maintainer or use GitHub's private vulnerability reporting feature
4. Include the following information:
   * Type of vulnerability
   * Full paths of source file(s) related to the vulnerability
   * Location of the affected source code (tag/branch/commit or direct URL)
   * Step-by-step instructions to reproduce the issue
   * Proof-of-concept or exploit code (if available)
   * Impact of the vulnerability
   * Any potential fixes you've identified

### What to Expect

* You should receive an acknowledgment within 48 hours
* We will investigate and validate the vulnerability
* We will work on a fix and keep you informed of our progress
* Once the vulnerability is fixed, we will publicly disclose it (with credit to you if desired)
* Critical vulnerabilities will be addressed as soon as possible

## Security Best Practices

When using this learning resource:

* Always use the latest version of Git and GitHub
* Follow GitHub's security best practices
* Never commit sensitive information (passwords, API keys, tokens)
* Use strong authentication methods (2FA recommended)
* Regularly review your repository's security settings

## Security Updates

Security updates will be released as needed. To stay informed:

* Watch this repository for updates
* Check the changelog for security-related changes
* Follow GitHub's security advisories

## Attribution

We appreciate responsible disclosure and will credit researchers who report valid vulnerabilities (unless they prefer to remain anonymous).

Thank you for helping keep this project and its users safe!
